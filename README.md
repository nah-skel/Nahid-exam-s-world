# Nahid Exam's World

সহজ ও ছোট একটি ক্লায়েন্ট‑সাইড (frontend-only) পরীক্ষা প্ল্যাটফর্ম — "Nahid Exam's World"।  
শিক্ষার্থী একটি আইডি দিয়ে পরীক্ষা দেবেন; একই আইডি দিয়ে একাধিকবার পরীক্ষা দেওয়া যাবে না। অ্যাডমিন সময়সীমা নিয়ন্ত্রণ, রেকর্ড দেখা, রফ করা (clear) ও CSV এক্সপোর্ট করতে পারবেন।

## ফিচারসমূহ
- শিক্ষার্থী আইডি দিয়ে পরীক্ষা শুরু করে উত্তর জমা দেয়।
- এক আইডি দিয়ে একবারই পরীক্ষা দেওয়ার সীমাবদ্ধতা (localStorage ভিত্তিক)।
- অ্যাডমিন কেবল পাসওয়ার্ড প্রদানের পরে:
  - ডিফল্ট পরীক্ষার সময় (মিনিট) সেট করতে পারবেন।
  - সব অ্যাটেমপট (রেকর্ড) দেখতে পারবেন।
  - সব রেকর্ড মুছতে পারবেন।
  - অ্যাটেমপট CSV হিসেবে এক্সপোর্ট করতে পারবেন।
  - প্রশ্নগুলো (answer সহ) দেখতে পারবেন — সেটা কেবল অ্যাডমিন লগইন করলে দেখাবে।
- সময়সীমা শেষ হলে স্বয়ংক্রিয়ভাবে পরীক্ষা জমা হবে।

## ফাইলসমূহ
- `index.html` — UI এবং অ্যাপ লেআউট
- `styles.css` — সিএসএস স্টাইলিং
- `script.js` — অ্যাপ লজিক (question list, timer, localStorage, admin features)
- `README.md` — এই ফাইল
- `.gitignore` — (ঐচ্ছিক)

## লোকালি চালানো
1. ফাইলগুলো এক ফোল্ডারে রাখুন: `index.html`, `styles.css`, `script.js`।
2. ব্রাউজারে `index.html` ডাবল‑ক্লিক করে খুলুন।
3. অ্যাডমিন (প্রথমে): Admin সেকশনে ডিফল্ট পাস ব্যবহার করে লগইন করে সময়সীমা সেট করুন (ডিফল্ট: `admin123`)।
4. শিক্ষার্থী: আইডি লিখে `পরীক্ষা শুরু করুন` ক্লিক করুন। পরীক্ষা চলাকালীন টাইমার দেখাবে। টাইমার শেষ হলে স্বয়ংক্রিয়ভাবে জমা হবে।

## GitHub Pages-এ হোস্ট করা
1. একটি পাবলিক রিপো তৈরি করুন (উদাহরণ: `nahid-exams-world`)।
2. উপরের ফাইলগুলো রিপো রুটে যোগ করুন এবং `main` ব্রাঞ্চে পুশ করুন।
3. GitHub → Settings → Pages → Branch = `main`, Folder = `/` নির্বাচন করুন এবং Save করুন।
4. কয়েক সেকেন্ড/মিনিটে আপনার সাইট লাইভ হবে: `https://<your-username>.github.io/<repo-name>`

## কিভাবে প্রশ্ন/পাস পরিবর্তন করবেন
- প্রশ্ন পরিবর্তন: `script.js`‑এর উপরের `QUESTIONS` অ্যারের ভিতরে প্রশ্ন/উত্তর বদলান।
- অ্যাডমিন পাস পরিবর্তন: `script.js`‑এ `const ADMIN_PASS = "admin123";` লাইনটি আপনার পছন্দের পাসে বদলান এবং রিপোতে commit & push করুন।

## সীমাবদ্ধতা ও নিরাপত্তা নোট
- সবকিছু client-side (ব্রাউজারে) চলছে এবং ডেটা localStorage‑এ রাখা হয়। তাই:
  - ব্যবহারকারী DevTools এ গিয়ে `script.js` থেকে প্রশ্ন/উত্তর দেখতে (বা localStorage কনটেন্ট পরিবর্তন) করতে পারবে।
  - ব্রাউজার ক্লিয়ার করা বা অন্য ডিভাইসে একই আইডি থেকে আবার পরীক্ষা দেওয়া সম্ভব।
- প্রোডাকশনে নিরাপদ করার জন্য সার্ভার‑সাইড ব্যাকএন্ড (authentication + persistent DB) লাগবে — আমি Firebase/Node+DB সেটআপ করিয়ে দিতে পারি যদি চান।

## অনুষঙ্গ/টিপস
- আগে বিস্তারিত টেস্ট করতে ব্রাউজারের DevTools → Application → Local Storage থেকে keyগুলো পর্যবেক্ষণ করুন।
- অ্যাডমিন কাজ করার পরে `Export CSV` দিয়ে রেকর্ড সংরক্ষণ করে নিন।
- যদি আপনি চান আমি README ইংরেজিতে বা অতিরিক্ত ডক যোগ করব, বলুন।
